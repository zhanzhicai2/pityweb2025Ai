## 项目概述

**Pity** 是一个功能强大的 API 测试平台，提供：

- 🔌 **接口测试**：支持 HTTP/HTTPS 接口测试和管理
- 📊 **测试计划**：灵活的测试用例组织和执行计划
- 📈 **测试报告**：详细的测试结果统计和历史记录
- 🔧 **测试配置**：环境管理、全局变量、数据库配置等
- 🔔 **消息通知**：支持邮件、钉钉、企业微信、飞书等多种通知方式
- 🛠️ **实用工具**：HTTP 测试、SQL 客户端、Redis 客户端等

## 技术栈

### 前端技术栈

- **框架**: React 18 + TypeScript
- **构建工具**: UmiJS 4.x (Max)
- **UI 组件**: Ant Design 5.x + Ant Design Pro Components
- **状态管理**: UmiJS 内置状态管理
- **图表**: @ant-design/charts, @antv/g6
- **编辑器**: react-ace (代码编辑)
- **其他**:
  - react-syntax-highlighter (语法高亮)
  - bytemd (Markdown 编辑器)
  - moment (时间处理)
  - lodash (工具函数)

### 后端技术栈

- **框架**: FastAPI (Python)
- **数据库**: MySQL (通过 SQLAlchemy ORM)
- **缓存**: Redis
- **异步**: aiomysql (异步 MySQL 驱动)
- **任务调度**: APScheduler
- **实时通信**: WebSocket
- **认证**: JWT + GitHub OAuth

## 核心概念

### 1. 用户角色系统

项目有三种用户角色：

- **普通用户 (role: 0)**: 基本权限
- **组长 (role: 1)**: 项目管理权限
- **超级管理员 (role: 2)**: 系统管理权限

### 2. 项目角色

在项目内部，用户有不同的角色：

- **负责人 (OWNER, role_id: 2)**: 项目所有者
- **组长 (ADMIN, role_id: 1)**: 项目管理员
- **组员 (MEMBER, role_id: 0)**: 普通成员

### 3. 测试用例状态

- **调试中 (status: 1)**: 正在开发和调试的用例
- **暂时关闭 (status: 2)**: 暂时不运行的用例
- **正常运行 (status: 3)**: 生产环境运行的用例

### 4. 用例构造类型

测试用例可以包含多种类型的步骤：

- **测试场景 (type: 0)**: 复杂的测试场景
- **SQL 语句 (type: 1)**: 数据库操作
- **Redis 命令 (type: 2)**: Redis 缓存操作
- **Python 方法 (type: 3)**: 自定义 Python 脚本
- **HTTP 请求 (type: 4)**: HTTP 接口调用

### 5. 断言类型

支持多种断言方式：

- `equal`: 等于
- `not_equal`: 不等于
- `in`: 包含于
- `contain`: 包含
- `length_eq`: 长度等于
- `json_equal`: JSON 等于
- 等等...（详见 src/consts/config.ts）

## 开发指南

### 代码规范

⚠️ **重要**: 2025 年 10 月 12 号之前写的代码不需要改动，尽量新建新文件，需要改动需要经过同意。

### 前端开发流程

1. **创建新页面**

   - 在 `src/pages/` 对应模块下创建组件
   - 在 `config/routes.ts` 中添加路由配置
   - 如需权限控制，添加 `authority` 字段

2. **调用后端 API**
   - 在 `src/services/` 中创建或添加 API 函数
   - 使用 `request` 发起请求
   - 通过 `auth.headers()` 添加认证头

示例：

```typescript
// src/services/myFeature.ts
import { request } from '@umijs/max';
import CONFIG from '@/consts/config';
import auth from '@/utils/auth';

export async function getData(params?: Record<string, string>) {
  return await request(`${CONFIG.URL}/myapi/data`, {
    method: 'GET',
    params,
    headers: auth.headers(),
  });
}
```

3. **使用状态管理**
   - 在 `src/models/` 中创建 model 文件
   - 使用 UmiJS 内置的状态管理

### 关键工具和组件

1. **Request 工具** (`src/utils/request`)

   - 统一的请求封装
   - 自动处理认证和错误

2. **Auth 工具** (`src/utils/auth`)

   - Token 管理
   - 用户信息获取
   - 权限验证

3. **公共组件** (`src/components/`)
   - CodeEditor: 代码编辑器
   - 各种业务组件

## 常见问题

### Q1: 如何修改后端 API 地址？

**A**: 编辑 `config/defaultSettings.ts` 文件，修改 `apiUrl` 和 `wssUrl` 字段。

### Q2: 如何添加新的路由？

**A**:

1. 在 `src/pages/` 下创建页面组件
2. 在 `config/routes.ts` 中添加路由配置

示例：

```javascript
{
  path: '/myfeature',
  name: '我的功能',
  icon: 'smile',
  component: './MyFeature/Index',
}
```

### Q3: 如何处理权限控制？

**A**:

1. 在路由配置中添加 `authority` 字段
2. 在组件中使用 `useAccess` 钩子检查权限

### Q4: 如何调试前端代码？

**A**:

1. 使用 Chrome DevTools
2. 安装 React Developer Tools 浏览器插件
3. 使用 `react-dev-inspector` (已集成)

### Q5: 如何添加新的 API 服务？

**A**:

1. 在 `src/services/` 下创建文件
2. 导出 API 函数
3. 在页面中导入使用

## 删除 node_modules 目录

1. 打开终端或命令提示符
2. 导航到项目根目录
3. 运行以下命令：

```bash
rm -rf node_modules
```

这将删除项目的 `node_modules` 目录，包含所有安装的依赖项。

重新安装依赖项：

```bash
npm install --legacy-peer-deps
```

或使用 Yarn：

```bash
yarn install
```

这将根据 `package.json` 或 `yarn.lock` 文件重新安装所有依赖项。
