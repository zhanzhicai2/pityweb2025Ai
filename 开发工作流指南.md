## 🐛 常见问题解决

### 问题 1: CORS 跨域错误

**错误信息：**

```
Access to XMLHttpRequest at 'http://localhost:7777/api/testcase'
from origin 'http://localhost:8000' has been blocked by CORS policy
```

**解决方案：**

```python
# 检查 app/__init__.py 中的CORS配置
pity.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 确保是 "*"
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 问题 2: 前端请求 404 错误

**检查清单：**

```bash
# 1. 确认后端服务是否启动
curl http://localhost:7777/api/testcase

# 2. 确认前端代理配置
# 查看 config/proxy.ts

# 3. 确认路由是否注册
# 查看 main.py 中的路由注册

# 4. 清除前端缓存
npm run start:dev
# 在浏览器中 Ctrl+Shift+R 强制刷新
```

### 问题 3: 数据类型不匹配

**常见原因：**

```python
# 后端期望的格式（下划线命名）
{
  "project_id": 1,
  "case_name": "测试用例"
}
```

```typescript
// 前端发送的格式（驼峰命名）
{
  projectId: 1,
  caseName: "测试用例"
}
```

**解决方案：**

在前端 API 服务中转换：

```typescript
export async function createTestCase(data: any) {
  // 转换为下划线命名
  const requestBody = {
    project_id: data.projectId,
    case_name: data.caseName,
  };

  return request('/api/testcase', {
    method: 'POST',
    data: requestBody,
  });
}
```

### 问题 4: 认证失败 401 错误

**检查清单：**

```typescript
// 确保请求包含认证token
import { getAuthToken } from '@/utils/auth';

export async function getTestCaseList() {
  return request('/api/testcase/list', {
    method: 'GET',
    headers: {
      Authorization: `Bearer ${getAuthToken()}`,
    },
  });
}
```

### 问题 5: 前端 build 后无法访问 API

**原因：** 生产环境代理配置不生效

**解决方案：**

```typescript
// 在构建前修改API地址
// 文件: src/consts/config.ts

export const API_BASE_URL =
  process.env.NODE_ENV === 'production'
    ? 'http://your-backend-server.com:7777' // 生产环境地址
    : ''; // 开发环境使用代理
```
